项目基础信息

    ● 项目类型：纯前端静态展示 Demo，无后端、无接口、无数据请求、无业务逻辑

    ● 页面数量：7 个（与项目原定页面完全一致，无增减）

    ● 开发适配：微信小程序原生静态页面，仅做视觉展示

    ● 核心要求：所有内容固定写死，仅实现基础布局与样式，无任何复杂交互

一、功能实现详细要求

页面 1：首页 - 树种选择页

    1.
    布局：顶部固定导航栏，居中显示标题 “认养一棵树”，右侧标注 “我的树” 文字入口；中间为竖向排列的树种卡片区域；底部居中标注 “静态演示版” 小字。

    2.
    固定内容：树种分类标签（果树类、观赏树类、公益树类）；5 张树种卡片，分别标注樱桃树 ¥99 / 年起、葡萄树 ¥129 / 年起、橘子树 ¥89 / 年起、樱花树 ¥159 / 年起、枫树 ¥119 / 年起。

    3.
    展示要求：卡片样式统一，文字内容固定，无点击、无滚动、无切换效果。

页面 2：认养信息填写页

    1.
    布局：顶部返回导航栏 + 页面标题 “树种认养”；中间为竖向表单区域；底部居中标注 “静态演示版” 小字。

    2.
    固定内容：两行输入框占位（您的昵称、给小树取名）；认养年限选项（1 年、3 年、5 年、10 年）；价格展示栏；底部固定 “确认认养” 按钮。

    3.
    展示要求：输入框仅为样式展示，无法输入；选项、按钮无点击效果，所有文字数值固定。

页面 3：我的树 - 生长交互页

    1.
    布局：顶部树状态展示栏；中间核心树模型展示区；下方四个操作按钮横向排列；底部居中标注 “静态演示版” 小字。

    2.
    固定内容：树名、健康值 100%、成长进度条；中间种子 / 小树静态展示区域；浇水、施肥 / 除虫、邀请共养、线下预约四个圆形按钮。

    3.
    展示要求：树模型、状态数据固定；按钮仅做样式展示，无任何点击、动画效果。

页面 4：树木照料服务付费页

    1.
    布局：顶部导航栏 + 标题 “小树专属照料”；中间竖向排列照料套餐卡片；底部居中标注 “静态演示版” 小字。

    2.
    固定内容：照料提示文字；三张套餐卡片（单次照料 ¥29、季度套餐 ¥79、全年套餐 ¥199）及对应服务说明；底部 “确认支付” 按钮。

    3.
    展示要求：套餐内容、价格固定；无选择、无支付跳转，纯静态展示。

页面 5：线下果园预约页

    1.
    布局：顶部导航栏 + 标题 “线下照料预约”；中间依次为日期选择区、时间段选择区、联系电话输入区；底部 “提交预约” 按钮 + 演示版小字。

    2.
    固定内容：日历、时间段选择按钮占位；联系电话输入框占位；预约规则提示文字。

    3.
    展示要求：无日期、时间选择功能，输入框无法输入，纯样式展示。

页面 6：共养邀请页

    1.
    布局：顶部导航栏 + 标题 “邀请共养”；中间小树信息卡片、共养成员列表、邀请卡片；底部演示版小字。

    2.
    固定内容：小树信息展示；“暂无共养好友” 文字提示；邀请文案 +“生成邀请海报” 按钮；共养规则说明文字。

    3.
    展示要求：无分享、无添加成员功能，按钮、文字均为静态展示。

页面 7：水果市集页

    1.
    布局：顶部导航栏 + 标题 “水果市集”；搜索框区域 + 搜索释义文字；我的摊位模块、两列商品卡片列表；右下角悬浮发布按钮；底部提示文字 + 演示版小字。

    2.
    固定内容：搜索框占位；搜索固定释义 “搜索，指仔细查找、搜寻，是主动寻找人、物或信息的行为”；我的摊位果实信息；4 张商品果实卡片；5% 摊位费提示文字。

    3.
    展示要求：搜索框无法输入，按钮、商品列表无点击效果，所有内容固定。

二、分步骤开发要求（共 3 步）

步骤 1：搭建基础页面框架

根据 7 个页面的布局要求，创建对应小程序页面文件，搭建基础 wxml 结构，编写页面骨架，划分导航栏、内容区、底部展示区，确定各区域层级与排版位置，所有结构代码固定，无动态渲染。

步骤 2：编写静态样式与填充内容

编写 wxss 样式代码，统一项目森系视觉风格（主色 #2E7D32、背景色 #F5F9F5），调整各模块间距、字体、卡片圆角；将各页面固定文字、按钮文字、提示内容直接写入页面，无任何动态数据绑定。

步骤 3：静态效果校验与整合

检查所有页面样式统一性、内容完整性，确认无任何交互逻辑、无点击事件、无数据请求；整合所有页面文件，确保可在微信开发者工具中正常预览，无报错、无冗余代码。

三、禁止开发项

    1.
    不开发任何后端交互、数据请求、表单提交、页面跳转功能

    2.
    不实现点击、动画、输入、选择等所有动态交互效果

    3.
    不添加图片、音频、3D 模型等外部资源，仅用文字与基础样式展示

    4.
    不开发登录、支付、分享、数据存储等业务功能
